# wing_flutter

## Физическая модель

Рассматривается тонкое эллиптическое тело из сетки `mesh/ellipse.msh`.
Тело считается абсолютно жёстким и подвешено за правую крайнюю точку большой
полуоси. В начальном положении большая полуось лежит вдоль `OX`, малая вдоль
`OY`, а вращение происходит в плоскости `XY` вокруг оси `OZ`, проходящей через
точку подвеса.

Ветер задан постоянным вектором

```text
U = (0, 8, 0)
```

то есть поток направлен вдоль `+OY`, перпендикулярно начальной большой полуоси.

Полный Navier-Stokes здесь не решается: в репозитории есть сетка самого тела,
но нет внешней расчётной области жидкости с входом, выходом и границами стенок.
Поэтому используется reduced-order FSI модель: давление и сила считаются
квазистационарно по относительной скорости потока, а движение тела описывается
одной степенью свободы - углом поворота вокруг шарнира.

## Кинематика тела

Для каждого узла хранится начальный радиус-вектор относительно шарнира:

```text
r0 = (relX, relY, relZ)
```

В момент времени `t` положение узла получается поворотом на угол `theta`:

```text
x = hingeX + relX cos(theta) - relY sin(theta)
y = hingeY + relX sin(theta) + relY cos(theta)
z = hingeZ + relZ
```

Линейная скорость узла считается как скорость точки жёсткого тела:

```text
v = omega x r
vx = -omega * ry
vy =  omega * rx
vz = 0
```

Здесь `omega = d theta / dt`, а `r = (rx, ry, 0)` - текущий радиус-вектор от
шарнира до узла.

В коде это реализовано в:

- `CalcMesh::updateKinematics()` - пересчёт координат узлов и поля `velocity`;
- `CalcMesh::configureWindFlutter()` - задание начального угла, ветра и
  параметров модели.

## Давление и сила

На каждом шаге для узла считается относительная скорость потока:

```text
u_rel = U - v_node
|u_rel| = sqrt(u_rel_x^2 + u_rel_y^2 + u_rel_z^2)
```

Динамическое давление:

```text
q = 0.5 * rho * |u_rel|^2
```

Чтобы давление было больше на стороне, откуда приходит ветер, вводится
безразмерный коэффициент наветренности `windward`. Он вычисляется через
проекцию узла на направление ветра:

```text
s = dot(x_node, U / |U|)
windward = (s_max - s) / (s_max - s_min)
pressureShape = 0.15 + 1.35 * windward^2
p = q * pressureShape
```

Узловая аэродинамическая сила направлена по относительному потоку:

```text
F_node = Cd * p * A_node * u_rel / |u_rel|
```

В коде это реализовано в:

- `CalcMesh::updateAerodynamicFields()` - расчёт `pressure`, `force`,
  `wind_velocity` и суммарного момента.

## Динамика вращения

Момент сил относительно шарнира считается суммой по узлам:

```text
Mz = sum(rx * Fy - ry * Fx)
```

Уравнение движения:

```text
I * domega/dt = Mz - c * omega
```

где:

- `I` - приближённый момент инерции сетки относительно шарнира;
- `c` - коэффициент углового демпфирования;
- `Mz` - момент аэродинамических сил.

Интегрирование выполняется явной схемой Эйлера:

```text
omega_next = omega + ((Mz - c * omega) / I) * dt
theta_next = theta + omega_next * dt
```

В коде это реализовано в:

- `CalcMesh::doTimeStep()` - один временной шаг: пересчёт полей, момент,
  угловое ускорение, новая угловая скорость и новый угол.

## Поля VTK

Каждый кадр экспортируется в `wing-step-*.vtu`, а временная коллекция - в
`wing.pvd`.

В ParaView доступны поля:

- `pressure` - скалярное давление в узлах;
- `velocity` - линейная скорость узлов тела от вращения вокруг подвеса;
- `force` - аэродинамическая сила, отнесённая к узлу;
- `wind_velocity` - заданная скорость ветра;
- `hinge_displacement` - смещение узла относительно начального положения;
- `body_state` в `FieldData` - `theta`, `omega`, `Mz`.

Экспорт реализован в:

- `CalcMesh::snapshot()` - запись одного `.vtu` кадра;
- `CalcMesh::writePvd()` - запись временной коллекции `.pvd`.
